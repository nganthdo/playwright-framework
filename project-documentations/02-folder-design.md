## Folder Design
### Purpose
- Structure the project helps maintainability and reusability clearly and better.

### Structure
```
src/
├── tests/ # Test cases (UI/API)
├── pages/ # Page Object Model (POM)
├── api/ # API 
├── fixtures/ # Test setup
├── utils/ # Shared helper functions
├── data/ # Test data
├── config/ # Environment configuration
```

### Explain
- **src**: source code directory contains project files.
- **tests**: directory for test case (UI/ API).
- **pages**: page object file representing different pages.
- **api**: directory for API related scripts.
- **fixtures**: directory for test fixtures, such as reusable functions.
- **utils**: directory for shared helper functions.
- **data**: directory for test data files.
- **config**: direcroty for environment configuration such as environment information, username/ password.
